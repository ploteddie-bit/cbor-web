CBOR-Web v3.0: eine Datei, Ihre gesamte Website

CBOR-Web erstellt eine strukturierte Kopie Ihrer Website, nativ für Maschinen konzipiert. Eine einzige Datei — index.cbor — im Stammverzeichnis Ihrer Domain enthält Ihren gesamten Inhalt: Seiten, Produkte, Preise, Navigation, alles in 7 Schlüsseln strukturiert, verifiziert, signiert.

Ihre HTML-Website bleibt für Ihre menschlichen Besucher identisch. CBOR-Web ist ein paralleler Kanal, unsichtbar für Internetnutzer, sofort erkennbar für jede KI.

Ein IETF-kompatibler Binärstandard

Eine einzige index.cbor-Datei, 7 Schlüssel, ~400 Zeilen Rust.

Das technische Problem

Jeder KI-Agent, der Ihre Website crawlt, führt die gleiche nutzlose Arbeit durch: Hunderte KB HTML + Assets herunterladen, das DOM parsen, CSS filtern, Metadaten deduplizieren (der Titel existiert 6-mal), 3 verschiedene Kodierungen verarbeiten. Um ~3 000 Bytes nützlichen Inhalt zu extrahieren.

Das ist kein KI-Problem. Es ist ein Formatproblem. HTML wurde 1993 für einen Menschen in einem Browser entwickelt.

Warum CBOR und nicht JSON?

JSON ist textuell, nicht-deterministisch, ohne Byte-Strings oder semantische Tags. CBOR ist ein IETF-Standard (STD 94) derselben Autoren wie JSON, verwendet in WebAuthn, COSE, CWT und IoT. Die deterministische Kodierung garantiert identische Bytes für denselben Inhalt — reproduzierbare Hashes, zuverlässige Caches.

Ökosystem-Kompatibilität

CBOR-Web ergänzt robots.txt, sitemap.xml, llms.txt, JSON-LD, OpenAPI. Es fügt die fehlende Schicht hinzu: vollständiger, strukturierter Inhalt in nativem Binärformat. Ein einziger GET statt eines mehrseitigen Crawls.
