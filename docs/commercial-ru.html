Реальная цена невидимости для ИИ

Ваш сайт красив. Он конвертирует. Он хорошо ранжируется в Google. Но когда клиент просит рекомендацию у ChatGPT, Claude или Gemini, ваша компания не появляется. Не потому что ваш контент плохой — потому что машины не могут его прочитать.

По данным Gartner (2025), объём поисковых запросов в традиционных поисковиках снизится на 25% к 2026 году в пользу ИИ-чатботов. Europanel/NIQ (2025) подтверждает, что 41% потребителей уже используют генеративный ИИ для поиска товаров. SEO для ИИ — это не SEO завтрашнего дня, а SEO сегодняшнего дня для канала, растущего на 305% в год (Cloudflare 2025, запросы GPTBot).

CBOR-Web v3.0: один файл, весь ваш сайт

CBOR-Web создаёт структурированную копию вашего сайта, изначально спроектированную для машин. Один файл — index.cbor — в корне вашего домена содержит весь контент: страницы, товары, цены, навигацию — всё структурировано в 7 ключах, проверено, подписано.

Ваш HTML-сайт остаётся неизменным для посетителей-людей. CBOR-Web — это параллельный канал, невидимый для пользователей, мгновенно обнаруживаемый любым ИИ.

Внедрение: 48 часов, ноль рисков

Никакой миграции, никакой переделки, никакого SEO-риска. Вы добавляете статический файл в корень.

  • Этап 1 (30 мин) — Сгенерировать index.cbor с метаданными. ИИ обнаруживает ваши страницы.
  • Этап 2 (2-4 ч) — Добавить контент страниц. ИИ читает весь ваш контент.
  • Этап 3 (1-2 дня) — Настроить навигацию, видимость, подпись. Полная индексация за 1 запрос.

Бинарный стандарт, совместимый с IETF

Один файл index.cbor, 7 ключей, ~400 строк на Rust.

Техническая проблема

Каждый ИИ-агент, сканирующий ваш сайт, выполняет одну и ту же бесполезную работу: загружает сотни Кб HTML + ассетов, парсит DOM, фильтрует CSS, дедуплицирует метаданные (заголовок существует 6 раз), обрабатывает 3 разные кодировки. Чтобы извлечь ~3 000 байт полезного контента.

Это не проблема ИИ. Это проблема формата. HTML был спроектирован в 1993 году для человека в браузере.

Интеграция на практике

  • Минимальная (30 мин) — Раздавать index.cbor с ключами 0-2-6. Как создать sitemap.xml.
  • Стандартная (1-2 дня) — Страницы (ключ 5) + навигация (ключ 4) + security (ключ 3). Полная индексация за 1 GET.
  • Полная (3-5 дней) — Генеративные блоки, шаблоны, тиры T0/T1/T2 с подписью COSE.

Почему CBOR, а не JSON?

JSON — текстовый, недетерминированный, без byte strings и семантических тегов. CBOR — стандарт IETF (STD 94) тех же авторов, что и JSON, используемый в WebAuthn, COSE, CWT и IoT. Детерминированное кодирование гарантирует идентичные байты для одного и того же контента — воспроизводимые хеши, надёжные кеши.

Совместимость с экосистемой

CBOR-Web дополняет robots.txt, sitemap.xml, llms.txt, JSON-LD, OpenAPI. Он добавляет недостающий слой: полный контент, структурированный, в нативном бинарном формате. Один GET вместо многостраничного краулинга.

Бизнес-модель — Тиры T2/T1/T0

T2 (Открытый): бесплатно, публичные страницы. T1 (Аутентифицированный): полный контент через token CBORW (ERC-20) или API key. T0 (Институциональный): верифицированные личности eIDAS 2.0.

Цикл органического принятия: ИИ-агенты фиксируют -94% затрат → предпочитают CBOR-Web → издатели внедряют → больше агентов предпочитают → сетевой эффект.

Драйверы роста

  • Сетевой эффект — Каждый принявший сайт увеличивает ценность для всех агентов.
  • Нулевая стоимость переключения — Добавляется к существующему. Никакой миграции.
  • Открытый стандарт, захватывающая экосистема — Протокол открыт, но инструменты и экономика токена создают ценность.
  • ESG-рычаг — 4,6 TWh/год сэкономлено (сценарий 10%) = измеримый аргумент.

Что решает CBOR-Web

  • Атрибуция — Каждый блок типизирован, структурирован, идентифицирован. Конец анонимной экстракции.
  • Качество чтения — Эмбеддинг на 100% сигнала точнее, чем эмбеддинг на 12% сигнала. Ваш контент лучше понимается, лучше рекомендуется.
  • Прямой доход — Модель T1 создаёт автоматический поток при каждом премиум-доступе. Без посредников.
  • Видимость для ИИ — Агент, выбирающий между 41 Кб HTML и 2,5 Кб CBOR, всегда выберет второе.
