O que um agente de IA realmente vê

Caso real: deltopide.fr — 679 linhas, 41 537 bytes. Medido em 25 de março de 2026.

Etapa 1 — O muro

41 Ko de HTML + 8 arquivos

O agente recebe ~260 Ko no total. CSS (30 Ko), JS de rastreamento (90 Ko), favicons, fontes. Para uma única página inicial.

Etapa 2 — O caos

4 formatos, 3 codificações

HTML + JSON-LD + JavaScript + CSS. Os acentos em 3 formas: &eacute; / \u00e9 / UTF-8. O título duplicado 6 vezes.

Etapa 3 — O ruído

88% de lixo

15+ divs vazias, 120 linhas de formulários, 80 linhas de JS, palavras cortadas ao meio para o estilo CSS.

Etapa 4 — O sinal

~3 000 bytes úteis extraídos

Após todo esse trabalho: o nome, as ofertas, os preços, o FAQ. 12% de sinal em 41 Ko recebidos. ~10 400 tokens consumidos.

2 487 bytes. 620 tokens. Pronto.

CBOR-Web v3.0: um único arquivo, todo o seu site

CBOR-Web cria uma cópia estruturada do seu site, projetada nativamente para máquinas. Um único arquivo — index.cbor — na raiz do seu domínio contém todo o seu conteúdo: páginas, produtos, preços, navegação, tudo estruturado em 7 chaves, verificado, assinado.

Seu site HTML permanece idêntico para seus visitantes humanos. CBOR-Web é um canal paralelo, invisível para os internautas, imediatamente detectável por qualquer IA.

O problema técnico

Cada agente de IA que rastreia seu site realiza o mesmo trabalho inútil: baixar centenas de Ko de HTML + assets, analisar o DOM, filtrar o CSS, deduplicar os metadados (o título existe 6 vezes), lidar com 3 codificações diferentes. Para extrair ~3 000 bytes de conteúdo útil.

Isso não é um problema de IA. É um problema de formato. HTML foi projetado em 1993 para um humano em um navegador.

Arquitetura CBOR-Web v3.0

Um único arquivo index.cbor na raiz. 7 chaves de primeiro nível:

ChaveConteúdoObrigatórioFunção
0 : @type"cbor-web"SimIdentificação
1 : @version3SimEvolução
2 : sitedomínio, nome, contato, geoSimMetadados
3 : securityauth, tiers T0/T1/T2, chaveSimControle de acesso
4 : navigationmenus, hierarquiaNãoEstrutura

Descoberta: GET /index.cbor + DNS TXT _cbor-web.dominio.tld opcional.

Integração na prática

  • Mínimo (30 min) — Servir um index.cbor com chaves 0-2-6. Como criar um sitemap.xml.
  • Padrão (1-2 dias) — Páginas (chave 5) + navegação (chave 4) + security (chave 3). Indexação completa em 1 GET.
  • Completo (3-5 dias) — Blocos generativos, templates, tiers T0/T1/T2 com assinatura COSE.

Por que CBOR e não JSON?

JSON é textual, não-determinístico, sem byte strings nem tags semânticas. CBOR é um padrão IETF (STD 94) dos mesmos autores do JSON, usado em WebAuthn, COSE, CWT e IoT. A codificação determinística garante bytes idênticos para o mesmo conteúdo — hashes reprodutíveis, caches confiáveis.

Compatibilidade com o ecossistema

CBOR-Web complementa robots.txt, sitemap.xml, llms.txt, JSON-LD, OpenAPI. Ele adiciona a camada que faltava: o conteúdo completo, estruturado, em binário nativo. Um único GET em vez de um crawl de múltiplas páginas.

O modelo de negócios — Tiers T2/T1/T0

T2 (Aberto): gratuito, páginas públicas. T1 (Autenticado): conteúdo completo via token CBORW (ERC-20) ou API key. T0 (Institucional): identidades verificadas eIDAS 2.0.

Ciclo de adoção orgânica: os agentes de IA constatam -94% de custos → preferem CBOR-Web → os publishers adotam → mais agentes preferem → efeito de rede.

Motores de crescimento

  • Efeito de rede — Cada site adotado aumenta o valor para todos os agentes.
  • Custo de mudança zero — Se adiciona ao existente. Zero migração.
  • Padrão aberto, ecossistema cativo — Protocolo aberto, mas ferramentas e economia do token geram valor.
  • Alavanca ESG — 4,6 TWh/ano economizados (cenário 10%) = argumento mensurável.

O paradoxo do editor em 2026

Você produz conteúdo de qualidade. Os agentes de IA fazem scraping do seu site para alimentar suas respostas. Sua expertise é redistribuída para milhões de usuários sem que você veja um centavo. O scraping é uma gambiarra onde a IA adivinha a estrutura a partir de seletores CSS que mudam a cada reformulação. Nenhum controle, nenhuma atribuição, nenhuma receita.
