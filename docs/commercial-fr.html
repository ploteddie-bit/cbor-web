~3 000 octets utiles extraits

Après tout ce travail : le nom, les offres, les prix, la FAQ. 12% de signal sur 41 Ko reçus. ~10 400 tokens consommés.

CBOR-Web v3.0 : un seul fichier, tout votre site

CBOR-Web crée une copie structurée de votre site, conçue nativement pour les machines. Un seul fichier — index.cbor — à la racine de votre domaine contient l'intégralité de votre contenu : pages, produits, prix, navigation, le tout structuré en 7 clés, vérifié, signé.

Votre site HTML reste identique pour vos visiteurs humains. CBOR-Web est un canal parallèle, invisible pour les internautes, immédiatement détectable par toute IA.

Le problème technique

Chaque agent IA qui crawle votre site effectue le même travail inutile : télécharger des centaines de Ko de HTML + assets, parser du DOM, filtrer le CSS, dédupliquer les métadonnées (le titre existe 6 fois), gérer 3 encodages différents. Pour extraire ~3 000 octets de contenu utile.

Ce n'est pas un problème d'IA. C'est un problème de format. HTML a été conçu en 1993 pour un humain dans un navigateur.

Intégration en pratique

  • Minimal (30 min) — Servir un index.cbor avec clés 0-2-6. Comme créer un sitemap.xml.
  • Standard (1-2 jours) — Pages (clé 5) + navigation (clé 4) + security (clé 3). Indexation complète en 1 GET.
  • Full (3-5 jours) — Blocs génératifs, templates, tiers T0/T1/T2 avec signature COSE.

Pourquoi CBOR et pas JSON ?

JSON est textuel, non-déterministe, sans byte strings ni tags sémantiques. CBOR est un standard IETF (STD 94) des mêmes auteurs que JSON, utilisé dans WebAuthn, COSE, CWT et l'IoT. L'encodage déterministe garantit des octets identiques pour le même contenu — hashes reproductibles, caches fiables.

Compatibilité écosystème

CBOR-Web complète robots.txt, sitemap.xml, llms.txt, JSON-LD, OpenAPI. Il ajoute la couche manquante : le contenu complet, structuré, en binaire natif. Un seul GET au lieu d'un crawl multi-page.

Le modèle économique — Tiers T2/T1/T0

T2 (Ouvert) : gratuit, pages publiques. T1 (Authentifié) : contenu complet via token CBORW (ERC-20) ou API key. T0 (Institutionnel) : identités vérifiées eIDAS 2.0.
