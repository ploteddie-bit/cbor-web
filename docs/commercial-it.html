Perché il vostro sito è invisibile per l'IA

E come risolvere il problema in 48 ore.

Il costo reale dell'invisibilità per l'IA

Il vostro sito è bello. Converte. È ben posizionato su Google. Ma quando un cliente chiede una raccomandazione a ChatGPT, Claude o Gemini, la vostra azienda non compare. Non perché il vostro contenuto sia scadente — perché le macchine non riescono a leggerlo.

Secondo Gartner (2025), il volume delle ricerche sui motori tradizionali calerà del 25% entro il 2026, a favore dei chatbot IA. Europanel/NIQ (2025) conferma che il 41% dei consumatori utilizza già l'IA generativa per ricerche di prodotti. Il SEO per l'IA non è il SEO di domani — è il SEO di oggi, per un canale che cresce del 305% all'anno (Cloudflare 2025, richieste GPTBot).

CBOR-Web v3.0: un solo file, tutto il vostro sito

CBOR-Web crea una copia strutturata del vostro sito, progettata nativamente per le macchine. Un solo file — index.cbor — alla radice del vostro dominio contiene l'intero contenuto: pagine, prodotti, prezzi, navigazione, il tutto strutturato in 7 chiavi, verificato, firmato.

Il vostro sito HTML resta identico per i visitatori umani. CBOR-Web è un canale parallelo, invisibile per gli utenti, immediatamente rilevabile da qualsiasi IA.

Implementazione: 48 ore, zero rischi

Nessuna migrazione, nessuna riprogettazione, nessun rischio SEO. Aggiungete un file statico alla radice.

  • Fase 1 (30 min) — Generare il vostro index.cbor con i metadati. Le IA scoprono le vostre pagine.

Il problema tecnico

Ogni agente IA che esplora il vostro sito esegue lo stesso lavoro inutile: scaricare centinaia di Ko di HTML + asset, analizzare il DOM, filtrare il CSS, deduplicare i metadati (il titolo esiste 6 volte), gestire 3 codifiche diverse. Per estrarre ~3 000 byte di contenuto utile.

Non è un problema di IA. È un problema di formato. HTML è stato progettato nel 1993 per un umano in un browser.

Perché CBOR e non JSON?

JSON è testuale, non deterministico, senza byte string né tag semantici. CBOR è uno standard IETF (STD 94) degli stessi autori di JSON, utilizzato in WebAuthn, COSE, CWT e IoT. La codifica deterministica garantisce byte identici per lo stesso contenuto — hash riproducibili, cache affidabili.

Compatibilità con l'ecosistema

CBOR-Web completa robots.txt, sitemap.xml, llms.txt, JSON-LD, OpenAPI. Aggiunge il livello mancante: il contenuto completo, strutturato, in binario nativo. Un solo GET invece di un crawl multi-pagina.

Il modello economico — Tier T2/T1/T0

T2 (Aperto): gratuito, pagine pubbliche. T1 (Autenticato): contenuto completo via token CBORW (ERC-20) o API key. T0 (Istituzionale): identità verificate eIDAS 2.0.

Ciclo di adozione organica: gli agenti IA constatano -94% di costi → preferiscono CBOR-Web → i publisher adottano → più agenti preferiscono → effetto rete.

Motori di crescita

  • Effetto rete — Ogni sito adottato aumenta il valore per tutti gli agenti.
  • Costo di switching nullo — Si aggiunge all'esistente. Zero migrazione.
  • Standard aperto, ecosistema captive — Protocollo aperto, ma strumenti ed economia del token creano valore.
  • Leva ESG — 4,6 TWh/anno risparmiati (scenario 10%) = argomento misurabile.
