Su sitio web es atractivo. Convierte. Está bien posicionado en Google. Pero cuando un cliente pide una recomendación a ChatGPT, Claude o Gemini, su empresa no aparece. No porque su contenido sea malo, sino porque las máquinas no saben leerlo.

CBOR-Web v3.0: un solo archivo, todo su sitio

CBOR-Web crea una copia estructurada de su sitio, diseñada nativamente para las máquinas. Un solo archivo — index.cbor — en la raíz de su dominio contiene todo su contenido: páginas, productos, precios, navegación, todo estructurado en 7 claves, verificado, firmado.

Su sitio HTML permanece idéntico para sus visitantes humanos. CBOR-Web es un canal paralelo, invisible para los internautas, inmediatamente detectable por cualquier IA.

El problema técnico

Cada agente IA que rastrea su sitio realiza el mismo trabajo inútil: descargar cientos de Ko de HTML + recursos, parsear el DOM, filtrar el CSS, deduplicar los metadatos (el título existe 6 veces), gestionar 3 codificaciones diferentes. Para extraer ~3 000 bytes de contenido útil.

Esto no es un problema de IA. Es un problema de formato. HTML fue diseñado en 1993 para un humano en un navegador.

¿Por qué CBOR y no JSON?

JSON es textual, no determinista, sin byte strings ni etiquetas semánticas. CBOR es un estándar IETF (STD 94) de los mismos autores que JSON, utilizado en WebAuthn, COSE, CWT y el IoT. La codificación determinista garantiza bytes idénticos para el mismo contenido — hashes reproducibles, cachés fiables.

Compatibilidad con el ecosistema

CBOR-Web complementa robots.txt, sitemap.xml, llms.txt, JSON-LD, OpenAPI. Añade la capa que faltaba: el contenido completo, estructurado, en binario nativo. Un solo GET en lugar de un rastreo multipágina.

El modelo de negocio — Niveles T2/T1/T0

T2 (Abierto): gratuito, páginas públicas. T1 (Autenticado): contenido completo vía token CBORW (ERC-20) o API key. T0 (Institucional): identidades verificadas eIDAS 2.0.

Bucle de adopción orgánica: los agentes IA constatan -94% de costes → prefieren CBOR-Web → los editores adoptan → más agentes prefieren → efecto de red.

Motores de crecimiento

  • Efecto de red — Cada sitio adoptado aumenta el valor para todos los agentes.
  • Coste de cambio nulo — Se añade a lo existente. Cero migración.
