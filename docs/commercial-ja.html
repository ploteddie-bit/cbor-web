AI 不可視の実際のコスト

あなたのサイトは美しく、コンバージョン率が高く、Google でも上位表示されています。しかし、顧客が ChatGPT、Claude、Gemini にレコメンドを求めると、あなたの企業は表示されません。コンテンツが悪いからではなく、機械が読めないからです。

Gartner (2025) によると、従来の検索エンジンの検索量は2026年までに25%減少し、AIチャットボットに置き換わります。Europanel/NIQ (2025) は、消費者の41%が既に商品検索に生成AIを使用していることを確認しています。AI検索最適化は明日のSEOではなく、今日のSEOです。このチャネルは年間305%成長しています（Cloudflare 2025、GPTBot リクエスト数）。

CBOR-Web v3.0：1ファイルであなたのサイト全体を

CBOR-Web は、機械向けにネイティブ設計されたサイトの構造化コピーを作成します。1つのファイル——index.cbor——をドメインのルートに置くだけで、ページ、商品、価格、ナビゲーションなどすべてのコンテンツを7つのキーで構造化し、検証・署名済みで提供します。

人間の訪問者向けのHTMLサイトはそのまま変わりません。CBOR-Web はインターネットユーザーには見えない並列チャネルですが、すべてのAIが即座に検出できます。

可視性オプション——AI が最初に見るものをあなたが制御

sitemap.xml の <priority> が Google に無視されるのとは異なり、CBOR-Web クローラーはあなたのシグナルを尊重します。

技術的な問題

あなたのサイトをクロールするすべてのAIエージェントが同じ無駄な作業を繰り返しています：数百KoのHTML+アセットのダウンロード、DOMの解析、CSSのフィルタリング、メタデータの重複排除（タイトルが6回存在）、3種類のエンコーディングの処理。そして抽出できるのは約3,000バイトの有用コンテンツだけ。

これはAIの問題ではありません。フォーマットの問題です。HTML は1993年にブラウザの人間ユーザー向けに設計されました。

ルートに index.cbor ファイルを1つ配置。7つのトップレベルキー：

  • フル（3〜5日） — 生成ブロック、テンプレート、T0/T1/T2 ティアと COSE 署名。

なぜ JSON ではなく CBOR なのか？

JSON はテキスト形式で非決定的、バイト文字列やセマンティックタグがありません。CBOR は JSON と同じ著者による IETF 標準（STD 94）で、WebAuthn、COSE、CWT、IoT で使用されています。決定的エンコーディングにより同一コンテンツは同一バイトを保証——再現可能なハッシュ、信頼性の高いキャッシュ。

エコシステム互換性

CBOR-Web は robots.txt、sitemap.xml、llms.txt、JSON-LD、OpenAPI を補完します。欠けていたレイヤーを追加：完全で構造化されたネイティブバイナリコンテンツ。マルチページクロールの代わりに1回の GET。

ビジネスモデル——T2/T1/T0 ティア

T2（オープン）：無料、公開ページ。T1（認証済み）：CBORW token（ERC-20）または API key で完全コンテンツにアクセス。T0（機関）：eIDAS 2.0 で身元確認済み。

オーガニック採用ループ：AI エージェントが94%のコスト削減を確認 → CBOR-Web を優先 → パブリッシャーが採用 → より多くのエージェントが優先 → ネットワーク効果。

成長エンジン

  • ネットワーク効果 — 採用するサイトが増えるほど、すべてのエージェントにとっての価値が上がる。
