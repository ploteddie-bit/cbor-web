CBOR-Web: Binary Web for AI Agents

Imaginez un monde où chaque agent IA peut lire l'intégralité du web en une seule requête. C'est ce que permet CBOR-Web : un format binaire standardisé qui réduit de 99% le bruit que les agents doivent traiter aujourd'hui.

Le web a été conçu pour les humains. HTML est un format de présentation — il décrit comment afficher l'information, pas ce qu'elle signifie. Quand un agent IA lit une page HTML, il doit télécharger et analyser des centaines de kilooctets de CSS, JavaScript, et balisage structurel pour extraire quelques paragraphes de contenu utile.

Le problème du gaspillage de tokens

Chaque page web contient en moyenne 7% de contenu utile. Les 93% restants sont du bruit : menus, pieds de page, publicités, scripts de tracking, bannières de cookies. Pour un agent LLM qui paie au token, cela représente un gaspillage massif.

  • Une page HTML type : 120 Ko
  • Contenu utile : ~8 Ko
  • Ratio signal/bruit : 7%
  • Tokens gaspillés par page : ~11 000

La solution CBOR-Web

CBOR-Web résout ce problème en permettant aux sites d'exposer une copie binaire de leur contenu, en parallèle du HTML existant. Les agents IA peuvent alors lire le contenu structuré directement, sans parsing HTML, sans DOM, sans gaspillage de tokens.

"The web has two clients: humans and machines. It's time to serve both."

Fonctionnalités clés

  • Format binaire CBOR (RFC 8949, standard IETF)
  • Réduction de 95-99% du volume de données
  • Contenu structuré natif (headings, paragraphes, listes, tableaux)
  • Découverte automatique via /.well-known/cbor-web
  • Mise à jour incrémentale via SHA-256
  • Bundle complet du site en une seule requête
