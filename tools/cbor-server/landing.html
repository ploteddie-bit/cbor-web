CBOR-Web

Binary web content format for autonomous AI agents. One file, one request, the entire website.

GitHub Live Demo

The Problem

AI agents crawl the web today by downloading HTML, parsing DOM trees, filtering scripts, ads, and CSS — wasting 88% of every token on noise. A typical 120 KB page delivers only 7% useful content.

The Solution

CBOR-Web adds a single binary file at the root of every website. An AI agent downloads it in one request and gets all the content — structured, typed, and ready to read. No HTML. No DOM. Just content.

120 KB HTML → 2 KB CBOR

Same information. Zero noise. Instant parsing.

How It Works

ComponentPurpose
/.well-known/cbor-webManifest — site metadata, page list, security
/.well-known/cbor-web/bundleFull site in one request
/.well-known/cbor-web/pages/*Individual pages with hashes
POST .../doleanceAgent → publisher feedback channel
GET .../diff?base=HASHIncremental updates (SHA-256)

Doléance Protocol

The web is a monologue today. CBOR-Web adds a return channel — agents tell publishers what was useful, what was noise, what was missing. Not analytics. Comprehension.
